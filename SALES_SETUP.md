# 売上表: 既存スプレッドシートから読み取る

画面（`sales.html`）は **閲覧のみ**。データは「いま使っている売上スプシ」から読みます。

## やること（3分）

### 1. 売上スプシのIDを控える

ブラウザで売上表を開き、URLを見る:

```
https://docs.google.com/spreadsheets/d/【この長い英数字がID】/edit...
```

### 2. シート（タブ）名を控える

例: `売上` `日次` `2026` など、**データが入っているタブ名**

### 3. GAS スクリプトプロパティに設定

シフト用 Apps Script → 歯車「プロジェクトの設定」→ スクリプト プロパティ:

| プロパティ | 値 |
|---|---|
| `SALES_SPREADSHEET_ID` | 手順1のID |
| `SALES_SHEET_NAME` | 手順2のタブ名 |
| `ADMIN_KEY` | （既存）`charge08` など |

日付・店舗の列名が特殊なときだけ:

| プロパティ | 例 |
|---|---|
| `SALES_DATE_HEADER` | `営業日` |
| `SALES_STORE_HEADER` | `店舗名` |

### 4. コード反映 + デプロイ

- `salesData` 一式を本番 GAS に入れる（`gas-sales-snippet.gs` または `gas-code-current.gs`）
- `doPost` に `case 'salesData': ...`
- **既存デプロイを新しいバージョンで更新**

### 5. 権限

- シフト用スプシの **⑥権限マスタ** に `ADMIN` 行があること（店舗ごと）
- 売上スプシは **GASを動かしているGoogleアカウントが閲覧できる** こと（共有）

## 列のルール

1行目＝ヘッダ。例:

| 日付 | 店舗 | 客数 | 売上 | … |
|------|------|------|------|---|

- **日付**: `日付` / `対象日` / `営業日` などを自動認識
- **店舗**: `店舗ID`（K1）または `店舗`/`店名`（店舗マスタの名前と照合）
- 他の列はすべて表に表示（数字列は合計しやすい）

## うまくいかないとき

| 表示 | 対処 |
|------|------|
| 売上シートが見つかりません | `SALES_SHEET_NAME` がタブ名と完全一致か |
| 日付列が見つかりません | 1行目のヘッダを確認し `SALES_DATE_HEADER` を設定 |
| 行が0件 | 日付がその月か、店舗名が①店舗マスタと一致するか |
| 不明なアクション salesData | GAS 未デプロイ |
| 権限エラーでシートを開けない | 売上スプシをGASのオーナーに共有 |

## 教えてもらえると設定を合わせやすい情報

1. スプシのURL（またはID）
2. タブ名
3. 1行目のヘッダ一覧（スクショ可）
